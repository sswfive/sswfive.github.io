## 什么是Docker？
> [Docker官网地址](https://www.docker.com/)

Docker是一款用Go语言开发的，用于开发、运行和部署应用程序的开放管理平台；它提供了一个在完全隔离环境中运行和打包应用程序的能力，通常将这个隔离的环境成为容器；
- 由于容器是隔离的环境，所以其具有隔离性和安全性；
- 可以在一个宿主机上同时运行多个容器，且不会相互干扰；


## Docker主要解决的什么问题？
- 保证了程序运行环境的一致性；
- 降低配置开发环境、生产环境的复杂度和成本；
- 实现程序的快速部署和分发；


## Docker的架构初探
Docker是一种C/S架构的产品（Docker Engine）。
- Server端：本质上一个守护进程（docker daemon）,主要的职责是负责创建并管理Docker的对象（镜像、容器、网络、数据卷）
- Client端：本质是一个命令行接口（Command Line Interface）,主要的职责是接收用户在命令行的指令，通过协议（REST API）直接操作Docker Daemon执行用户操作。
- REST API: 一套用于和Docker Daemon通信的接口协议；

Docker的核心组件：
- Docker Client（客户端）
	- 用户与Docker的主要交互方式
- Docker Server（服务端）
	- 负责监听API请求
- Docker Registries（仓库）
	- 用于存储镜像的云服务环境，比较有代表性的就是：Docker Hub
- Docker Objects（对象）
	- Image（镜像）
	- Container（容器）
	- Network（网络）
	- Volume（数据卷）

## Docker的的核心实现
Docker主要是基于几个Linux的内核特性来实现的。
- Namesapaces（Linux的命名空间）
- CGroups（Control Groups: Linux的控制组）
- UFS(Union File Systems: Linux的联合文件系统)

> Docker原则上只能运行在Linux上，但为啥Win、MacOS都有对应的安装版本呢，其本质上是借助了虚拟化技术，然后在Linux虚拟机运行Docker程序。


### Namespaces的职责：
- 为Docker容器提供操作系统层面的隔离，具体的隔离体现如下：
	- 进程号隔离：每一个容器内运行的第一个进程，进程号总是从1开始算；
	- 网络隔离：容器的网络与宿主机或其他容器的网络是隔离的，分开的，相当于两个网络；
	- 进程间通信隔离：容器中的进程与宿主机或其他容器中的进程是相互不可见的，通信需要借助网络；
	- 文件系统挂载隔离：容器拥有自己单独的工作目录；
	- 内核及系统版本号隔离：容器内查看内核版本号实质上查看的是容器的，而不是宿主机；


### CGroups的职责：
- 为Docker容器提供硬件层面的隔离，具体的隔离体现如下：
	- 控制组能控制应用程序所有的硬件资源；
	- 基于该特性，控制组帮助Docker Engine 将硬件资源共享给其他容器使用，并加以约束限制，如控制容器所使用的内存大小等等；


### UFS的职责
- Union File Ststems (联合文件系统),利用分层（Layer）思想管理镜像和容器；
